Data Analytics Case Study

Finding KPI drivers with EDA + cohorts

A repeatable workflow for decision-ready insights: define the question, validate data, analyze cohorts/segments, and deliver recommendations with measurable success criteria.

Tools SQL + Excel + Python Methods cohorts / segmentation Output KPI narrative

1) Frame the decision

  • What decision will change if we know the answer?
  • Define the KPI and the time window.
  • List hypotheses (what might be driving change).

Outputs

  • Success metric(s) + KPI definition
  • Segment list (channel, plan, cohort, geo, etc.)
  • Questions prioritized by impact

2) Validate data

Trust comes from reconciliation and clear metric grain.

Validation checklist

  • Row counts and missingness
  • Reconcile against a known source of truth
  • Outlier review (not removal by default)
  • Confirm metric grain (user/day/order…)

Cohort SQL pattern (template)

-- Cohort template (anonymized)
WITH first_event AS (
  SELECT user_id, MIN(event_date) AS cohort_date
  FROM events
  GROUP BY 1
),
activity AS (
  SELECT e.user_id, f.cohort_date, e.event_date
  FROM events e
  JOIN first_event f USING(user_id)
)
SELECT cohort_date,
       DATE_DIFF(event_date, cohort_date, DAY) AS day_n,
       COUNT(DISTINCT user_id) AS users
FROM activity
GROUP BY 1,2
ORDER BY 1,2;

Replace with your real query pattern (keep it anonymized).

3) Analyze + explain

Trends, cohorts, and driver analysis that leads to action.

Analysis outputs

  • Trend + seasonality context
  • Segment differences and what explains them
  • Driver prioritization (impact × effort)

Recommendation format

  • Action: what to change
  • Expected effect: KPI lift estimate
  • Measure: success metric + time window

Deliverable package

  • 1-page executive summary
  • KPI dashboard/report
  • Decision log + next steps
